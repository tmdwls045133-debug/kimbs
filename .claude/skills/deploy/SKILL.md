# 배포 스킬

git 저장소의 모든 변경사항을 자동으로 커밋하고 푸시합니다.

## 워크플로우

```
1. git status 실행 → 변경사항 파악
   ├─ untracked files
   ├─ modified files
   └─ deleted files

2. 변경 파일 목록 출력
   └─ 각 파일별 상태 표시 (추가됨/수정됨/삭제됨)

3. git add . 로 모든 변경사항 스테이징

4. 커밋 메시지 생성
   └─ 형식: "Update: [YYYY-MM-DD] - [변경사항 요약]"

5. git commit 실행

6. git push origin master 실행

7. 완료 리포트 출력
   ├─ 커밋 해시
   ├─ 파일 수
   ├─ 푸시 결과
   └─ 상태 메시지
```

## 구현

다음 단계를 순서대로 실행합니다:

### 1단계: 현재 위치 확인
현재 작업 디렉토리가 git 저장소인지 확인합니다.

### 2단계: git status 확인
```bash
cd [repo-path] && git status --porcelain
```

### 3단계: 변경사항 파악 및 출력
- 각 라인을 파싱하여 파일 상태 분류
- untracked (??), modified (M), deleted (D) 등 표시

### 4단계: 변경사항 있는지 확인
- 변경사항이 없으면 "변경사항 없음" 메시지 출력 후 종료
- 변경사항이 있으면 계속 진행

### 5단계: 변경사항 스테이징
```bash
cd [repo-path] && git add .
```

### 6단계: 커밋 메시지 생성
- 현재 날짜 취득 (YYYY-MM-DD 형식)
- 변경된 파일 수 계산
- 메시지: "Update: [날짜] - [파일수]개 파일 변경"

### 7단계: 커밋 실행
```bash
cd [repo-path] && git commit -m "[메시지]"
```

### 8단계: 커밋 정보 수집
```bash
cd [repo-path] && git log -1 --pretty=format:"%H %s %ar"
```
커밋 해시, 메시지, 상대 시간 추출

### 9단계: 푸시 실행
```bash
cd [repo-path] && git push origin master
```

### 10단계: 완료 리포트 출력
```
✓ 배포 완료!
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
커밋: [해시 (처음 7자)]
메시지: [커밋 메시지]
변경 파일: [개수]개
푸시 상태: ✓ 성공
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 주의사항

- 현재 브랜치가 master여야 합니다
- 원격 저장소(origin)가 설정되어 있어야 합니다
- 스테이징할 파일이 없으면 커밋하지 않습니다
- 푸시 실패 시 오류 메시지를 표시합니다

## 변경사항이 없는 경우

```
ℹ 변경사항 없음 - 배포할 파일이 없습니다
```

## 오류 처리

- git 명령 실패 시: 오류 메시지와 함께 종료
- 푸시 실패 시: "푸시 실패" 표시하고 오류 메시지 출력

## 스킬 실행

이 스킬은 번들된 `scripts/deploy.sh` 스크립트를 사용합니다. 사용자가 `/배포` 명령어를 실행하면:

```bash
bash [skill-path]/scripts/deploy.sh [current-repo-path]
```

스크립트는 자동으로:
1. 현재 git 저장소의 변경사항을 확인
2. 모든 변경사항을 스테이징하고 커밋
3. master 브랜치에 푸시
4. 완료 리포트 출력
